Build a complete Swiss Ephemeris–based Vedic astrology web app with both front end and API endpoint using FastAPI and pyswisseph.

Core Features:

Front End UI (Single Page App):

Collect user inputs: Date of Birth, Time of Birth, Birthplace Latitude, Longitude, Timezone, and Ayanamsha choice (dropdown with common options like Lahiri, Raman, Krishnamurti).

A Submit button should:

Display the results below the form: Ascendant, full decimal longitudes of the 9 Vedic planets, and the Ayanamsha value used (in degrees° minutes′ seconds″).

Show the API endpoint URL with a button to open the raw API response in a new tab.

Provide a “Copy All Longitudes” button to copy all numeric values at once.

API Endpoint (/chart):

Accept both GET and POST methods.

Parameters: year, month, day, hour (decimal UT), lat, lon, tz, ayanamsha.

Return JSON with: julian_day_ut, chosen ayanamsha_name, ayanamsha_value_dms, ascendant_deg, and a dictionary planets_deg with full decimal precision.

Admin Features:

Admin login page with username/password.

Admin can:

Set Authorized Domains that can access the API.

Generate & manage API keys.

The API must validate incoming requests: only allow calls from authorized domains or with a valid API key.

Implementation Details:

Use FastAPI for backend logic and API routes.

Use vanilla JS, HTML/CSS, or a minimal frontend framework (e.g., Vue or React) for the UI. Keep it simple to fit Replit’s free plan.

Use uvicorn run command:

uvicorn main:app --host 0.0.0.0 --port 8080


Place Swiss ephemeris .se1 files in an ephe folder and set the ephemeris path accordingly.

Make sure to set the sidereal mode according to the chosen ayanamsha (e.g., Lahiri = swe.SIDM_LAHIRI).
